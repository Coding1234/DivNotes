Inline vs Block Elements

HTML elements are either inline or block. Block elements form an "invisible box" around the content inside of them, leaving a space before and after the content. (this space can be adjusted), Inline elements dont do anything to the line.

DRY

The principle of DRY (don't repeat yourself) was formulated by Andy Hunt and Dave Thomas in The Pragmatic Programmer. This principle stresses that a programmer should write code that allows them to write the logic once and use or invoke it as many times as needed. For example in my CSS file I have styled the lesson class in a particular way. This is done to ensure that all div's with the lesson class get a consistent look and feel. The same effect can be accomplished by styling each individual div tag in the HTML document however this requires duplication of style information for each tag involved. More typing means more potential errors and a larger HTML file which is not good. The CSS file provides a single set of instructions for divs named 'class'. It is a good way to accomplish the styling with fewer typed characters. Changes to the styling can be made in a single location and be automatically reflected across the entire HTML document this saves time and reduces errors.

css stands for "cascading style sheets". it helps programmers aboid repitition. when using CSS the style of the outermost tag will apply to all the children tags but the properties will not neccisarily.instead of using things like the b tag in your HTML file, you can use tags like font weight in CSS whiuch gives you a greater power over things like "how bold you want SAMPLE to be" etc.

CSS vocab.

Style: This word can refer to many things and so it can be confusing. It can refer to:
  • The HTML Tag: for example
  • The HTML attribute. For example:
    THIS TEXT IS NOW PINK
  • The general look of a web page. For example: "I like that CSS style/styling."
  • A verb. For example: "all i need to do in order to finish this webpage is style it (yay!)."

Rule: a line of CSS code describing the value that a certain attribute should take.
Property: The property you want to change.
Value: The value that you want to assign to the attribute.
Selector: The name that you use to in order to target the elements that are assigned to a class or id attribute in the HTML.
Class: A class refers to a group of elements that can be styled together. (Class names should not contain periods or any other punctuation marks such as class="1.1")
ID: ID's are unique identifiers that uniquely identifies an element in HTML.
in HTML, code comments begin with . Everything in between is ignored, whereas in CSS, code comments begin with /* and end with */.
